Concept 01

console.log()

console.log() is JavaScript's way to print a value. It sends output to the developer console so you can see what your code is doing.

script.js
console.log("Hello World"); console.log(5); console.log("I am learning JavaScript");
Console output
Hello World
5
I am learning JavaScript
Text values — strings
Text goes inside quotes.
"Hello World"  or  'Hello World'
Number values
Numbers go in directly — no quotes needed.
5  3.14  100
Concept 02

Variables with let

A variable is a named container that holds a value. Instead of repeating the same value everywhere, you give it a name and use that name.

script.js
let name = "Bilal"; let age = 20; console.log(name); // Bilal console.log(age); // 20
Keyword — let
Tells JavaScript: create a new variable. Write it only once — when you first declare the variable.
Naming convention
Use descriptive names in camelCase for multi-word names:
firstName  totalScore  itemPrice
Concept 02 — continued

Variables can change

A variable can be updated after it is created. You do not write let again — just use the variable name by itself.

script.js
let score = 10; console.log(score); // 10 score = 15; // update — no "let" here console.log(score); // 15
⚡ Quick question
Why don't we write let score = 15; on the second assignment?
Because let creates a variable. The box already exists — we are just putting a new value inside it. Writing let again would cause an error: "already been declared".

Mini game 02  ·  Fix the order

Arrange the lines correctly

These four lines are shuffled. What is the correct order so the code runs without errors?

Aconsole.log(total);
Blet price = 10;
Clet quantity = 3;
Dlet total = price * quantity;
correct — B → C → D → A
let price = 10; let quantity = 3; let total = price * quantity; console.log(total); // 30
JavaScript reads code top to bottom. You must declare a variable before you can use it.
